# Mimir V12 Design

Date: 2026-04-25

## Goals

1. Reduce `/mimir-update` token cost (~200 â†’ ~30 tokens)
2. Add `--output json` for machine-readable estimates
3. Replace flat `800 tok/turn` constant with real per-project average from transcript
4. Gemini CLI compatibility (mirrors `codex/` structure)

---

## Feature 1 â€” Token-free update

### Problem

`mimir-update.md` skill file is ~200 tokens loaded into context each time the command runs.

### Solution

Split into thin slash command + standalone shell script.

**`scripts/update.sh`** (new file in repo):
```bash
#!/bin/bash
set -e
if [ -d ~/.claude/mimir/.git ]; then
  git -C ~/.claude/mimir pull --ff-only
else
  rm -rf ~/.claude/mimir
  git clone https://github.com/ZonatedCord/Mimir.git ~/.claude/mimir
fi
mkdir -p ~/.claude/commands/
cp -r ~/.claude/mimir/.claude/commands/* ~/.claude/commands/
echo "âś… Mimir updated"
```

**`.claude/commands/mimir-update.md`** (replace current content):
```markdown
---
description: Update Mimir
---
Execute verbatim: bash ~/.claude/mimir/scripts/update.sh
```

Side-effect: `git pull` is faster than rm+clone (no re-download of full history).

After this change, `cp -r scripts/* ~/.claude/mimir/scripts/` sync step must include `update.sh`.

## Feature 2 â€” `--output json`

### Change: `scripts/estimate.js`

Add `--output json` to `parseArgs`. Return `{ outputJson: true }` when flag present.

In `main()`: when `outputJson` is true, skip all formatted output and write one JSON object to stdout:

```json
{
  "task": "string",
  "totalTokens": 12400,
  "taskTokens": 800,
  "contextTokens": 11600,
  "risk": "MEDIUM",
  "suggestedModel": "claude-sonnet-4-5",
  "headroomPct": 88,
  "method": "exact|heuristic",
  "files": [
    { "label": "context.js", "tokens": 420, "auto": true }
  ]
}
```

Behavior in JSON mode:
- History still written via `appendHistory()`
- Split suggestions suppressed
- `process.exit(0)` after JSON write, no extra newlines

## Feature 3 â€” Real tokens/turn from transcript

### Problem

`TOKENS_PER_TURN = 800` is a flat constant. Real conversations often run 1,200â€“2,000 tok/turn depending on model and task verbosity.

### Solution

New function `readActualTokensPerTurn(cwd)` in `scripts/lib/context.js`.

**Algorithm:**
1. Encode project path: `cwd.replace(/^\//, '').replace(/\//g, '-')` â†’ e.g. `Users-marcobarlera-Documents-02-PROGETTI-Mimir`
2. Find conversations dir: `~/.claude/projects/<encoded>/conversations/`
3. List `.jsonl` files, sort by mtime descending, take up to 5 most recent
4. For each file, read lines, parse JSON, collect `content` of entries where `role === 'assistant'`
5. Count tokens via `estimateTokens(content)` for each assistant message
6. Return `Math.round(average)` across all assistant messages found
7. Fall back to `TOKENS_PER_TURN` (800) if: dir missing, no files, parse errors, fewer than 3 messages

`estimateContextOverhead()` calls `readActualTokensPerTurn(cwd)` instead of `TOKENS_PER_TURN` when `turns > 0`.

**Display line** (in `estimate.js`):
- With real data: `Conversation (5 turns):  ~6,200  (~1,240 tok/turn, from transcript)`
- Fallback: `Conversation (5 turns):  ~4,000  (~800 tok/turn)`

`estimateContextOverhead` returns `tokensPerTurn` and `tokensPerTurnSource` (`'transcript'|'default'`) in its result for use by estimate.js display.

### JSONL format (verified)

Each line is a JSON object. Filter: `entry.type === 'assistant'`. Content lives at `entry.message.content` â€” always an array of blocks. Count only blocks where `block.type === 'text'`; skip `thinking`, `tool_use`, `tool_result`.

```json
{
  "type": "assistant",
  "message": {
    "role": "assistant",
    "content": [
      { "type": "thinking", "thinking": "..." },
      { "type": "text", "text": "actual response text" },
      { "type": "tool_use", "name": "Bash", "input": {} }
    ]
  }
}
```

Path encoding: `cwd.replace(/[^a-zA-Z0-9]/g, '-')` â€” replaces `/`, `_`, spaces with `-`. Example: `/Users/foo/bar_baz` â†’ `-Users-foo-bar-baz`.

## Feature 4 â€” Gemini CLI compatibility

### Structure

```
gemini/
  GEMINI.md
  skills/
    mimir.md
    mimir-diff.md
    mimir-help.md
    mimir-config.md
    mimir-history.md
    mimir-update.md
```

`GEMINI.md` is auto-loaded by Gemini CLI from the project root. It points to the skills dir.

Skills adapt slash command syntax to Gemini's `activate_skill` conventions. Start from `codex/skills/` as templates; replace Codex-specific instructions (tool names, hook paths) with Gemini equivalents.

**Install step** (documented in README):
```bash
mkdir -p ~/.gemini/skills
ln -sf ~/.claude/mimir/gemini/skills ~/.gemini/skills/mimir
```

### `gemini/GEMINI.md` content

Minimal: explains Mimir, lists available skills, shows example usage in Gemini CLI context.
